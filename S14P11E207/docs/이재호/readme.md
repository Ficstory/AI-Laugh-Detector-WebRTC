## PM으로서 배운 점

### 1. 가설 검증의 중요성
컨설턴트 미팅에서 "토이 프로젝트" 우려를 받았을 때, 단순히 기능을 나열하는 것이 아닌 **실제 사용자 니즈 검증**이 필요함을 깨달았다. 짧은 프로젝트 기간을 고려하여 빠르게 설문을 설계하고 배포하였다. 그 결과 48시간 내 168개의 설문 응답을 확보하여 데이터 기반 의사결정을 할 수 있었다.

### 2. 빠른 피봇과 유연한 대응
설문 결과 "얼굴 노출 부담"이라는 예상치 못한 Pain Point가 드러났다. 이를 바탕으로 MVP를 재조정하고 친구/지인 매칭을 메인으로 전환하는 결정을 빠르게 내렸다.

### 3. 문서화와 커뮤니케이션
- 모든 회의 내용을 회의록으로 기록하여 의사결정 근거 보존
- 개별 시장조사를 통합 보고서로 정리하여 팀 전체 인사이트 공유
- 컨설턴트 피드백을 액션 아이템으로 즉시 전환

### 4. 기술적 리스크 관리
컨설턴트 미팅을 통해 파악한 주요 기술 과제:
- 실시간 웃음 감지 모델의 응답 속도
- 화상 연결 안정성 (OpenVidu 포트 이슈)
- 보안 문제 (캡처/녹화 방지)

이를 기능명세서에 우선순위로 반영하여 개발 단계에서의 병목을 예방하고자 했다.

### 5. 일정 관리
1월 13일 기획 확정이라는 타이트한 일정 속에서:
- 주말(10~11일) 시장조사 병행
- 12일 컨설팅 직후 설문 설계 착수
- 13일 오전 배포 → 14일 오후 마감으로 빠른 데이터 수집

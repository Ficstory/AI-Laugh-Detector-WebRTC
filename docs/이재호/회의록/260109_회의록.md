## 안건별 논의 내용

### 안건 1: 아이디어 구체화

**논의 배경**

개그 배틀 아이디어를 구체화하고 필요한 기능을 파악한다.

**논의 내용**

**양한빈 아이디어:**
- 미세 표정 변화 감지 시스템 (웃음 참기 게이지)
- 하이라이트 자동 생성 및 아카이빙
- 데스매치 시 상금 제도

**이재호 아이디어:**
- 로그인/로그아웃
- 1:1 매칭 및 다대일(4명/8명) 매칭
- 웃음 인식 모델
- 랭킹 시스템 (승리 횟수 기반)
- 신고 시스템, 관전 시스템, 실시간 채팅
- 영상/음성 필터, 웃음 기준 온보딩, 턴제 룰

**차경빈 아이디어:**
- 도네이션 기능
- 채팅 비속어 필터

**김승철 아이디어:**
- 웃음 소리도 패배 조건으로 추가
- 자기 차례에는 웃음 규칙 완화
- 관전자 투표

**박세홍 아이디어:**
- 데스매치 모드

**오언서 아이디어:**
- 웃은 사람 하이라이트 + 생존 투표
- 뿅망치 클릭 이벤트

**유준호 아이디어:**
- 랜덤매칭 (턴제/토크온 방식)
- 준비시간 3~5초 필요
- 다대다 방 직접 입장

**결정 사항 (Decision)**

✅ 핵심 기능 목록 확정 (우선순위는 MVP 논의에서 결정)

**결정 근거**

팀원 전체 아이디어를 종합하여 서비스 완성도 높은 기능 세트 구성

### 안건 2: MVP 선정

**논의 배경**

제한된 개발 기간 내 핵심 기능에 집중하기 위한 MVP 범위 설정 필요

**논의 내용**

- 로그인 방식: 쿠키 vs 세션 vs JWT → 소셜로그인(카카오) 선택
- 화상회의 라이브러리 조사 필요
- 웃음 감지 모델 서칭 담당: 양한빈
- 반응형 여부: MVP 단계에서는 웹버전만 구현

**결정 사항 (Decision)**

✅ MVP 기능
- 소셜 로그인 (카카오)
- 1:1 매칭
- 웃음 감지 모델
- CI/CD (백엔드 우선)

✅ MVP 단계 제외
- 반응형 (웹버전만 구현)
- 디자인은 웹/모바일 2개, 구현은 웹 1개

**결정 근거**

핵심 차별점인 '웃음 감지 + 화상 대결'에 집중하여 빠른 프로토타입 제작

### 안건 3: 피드백 반영

**논의 배경**

팀 내부 피드백을 통한 서비스 방향 조정

**논의 내용**

- 턴제이긴 하지만 진행을 도와주는 기능 필요
- 너무 열린 서비스는 취지가 흐려질 수 있음
- 준비 버튼 추가 필요

**결정 사항 (Decision)**

✅ 준비 버튼 추가
✅ 게임 진행 가이드 기능 검토

**결정 근거**

사용자 경험 개선 및 서비스 취지 명확화
